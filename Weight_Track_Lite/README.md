# WeightTrack Lite

## CS 360 Mobile Architecture and Programming  
**Project Three – Mobile App Portfolio Artifact**

This repository contains the Project Three mobile application and portfolio artifact for CS 360 Mobile Architecture and Programming. The app was developed using Android Studio and demonstrates user-centered mobile design, structured app development, and practical Android programming concepts.

### App Requirements and Goals
The goal of this project was to design and develop a functional Android mobile application using user-centered design principles. The app was created to meet user needs through a clear interface, intuitive navigation, and responsible handling of user data and permissions. Project Three built on the UI designs created earlier in the course and required translating those designs into a working application.

### Screens Necessary for a User-Centered UI

- **Login Screen**
  - Allows returning users to securely sign in.
  - Designed with a simple layout, clear input fields, and an obvious primary action to reduce friction.

- **Create Account / Registration Screen**
  - Enables new users to create an account within the app.
  - Keeps required fields minimal and clearly labeled to avoid confusion.

- **Main Dashboard / Home Screen**
  - Displays key information such as current weight, goal weight, and progress.
  - Designed to give users an at-a-glance overview without needing to navigate through multiple screens.

- **Add / Log Weight Screen**
  - Allows users to enter and save daily weight entries.
  - Focused on a single task with minimal input to make logging quick and easy.

- **Weight History Screen**
  - Shows previously logged weight entries in an organized list.
  - Helps users track trends and confirms that their data is being stored correctly.

- **Set Goal Weight Screen**
  - Allows users to set or update a target goal weight.
  - Designed to be simple and flexible so users can adjust goals as needed.

- **Notifications / SMS Settings Screen**
  - Allows users to enable or disable SMS notifications.
  - Designed to respect user permissions, with the app remaining functional even if notifications are declined.

- **User Profile / Account Screen**
  - Provides basic account management options such as viewing account information or logging out.
  - Offers a predictable location for user-related actions.

### Why the UI Design Was Successful

- Screens were designed with **one primary purpose each**, reducing cognitive overload.
- Navigation follows a **logical and consistent flow**, making the app easy to learn.
- Layouts emphasize **clarity, readability, and accessibility**.
- Permission handling is transparent and optional, keeping users in control.



### Coding Approach and Development Strategy
Development was completed in Android Studio using an incremental approach. Features were implemented and tested in small sections, which made debugging more manageable. Working extensively in Android Studio helped me start seeing the full picture of mobile app development and how UI design, app logic, and data handling connect. Many of these strategies can be applied to future mobile and software development projects.

### Testing and Validation
Testing was performed throughout development by frequently running the app in the emulator and verifying functionality after changes were made. This process was important for ensuring the app behaved as expected and helped identify issues early. Testing also highlighted how small changes can impact overall usability.

### Design Challenges and Areas of Strength
One challenge during this project was managing scope while still meeting all requirements. It was important to focus on core functionality rather than adding unnecessary features. I was particularly successful in translating UI designs into a functional Android application and applying concepts from previous courses, such as planning, user flow, and data management, within a mobile development environment.
